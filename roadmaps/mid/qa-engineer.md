# Mid QA Engineer

## Description

What a mid-level QA engineer should know — test automation, CI integration, test strategy, and mentoring junior testers.

## Prerequisites

- [Junior QA Engineer](../junior/qa-engineer.md)

## Learning Path

### Test Automation

- `🔴 CRITICAL` E2E automation — Playwright, Cypress, or Selenium
- `🔴 CRITICAL` Page Object Model — structuring automation code
- `🔴 CRITICAL` API automation — REST API testing with code (Supertest, pytest, REST Assured)
- `🟠 HIGH` Data-driven testing — parameterized test cases
- `🟠 HIGH` Mobile automation — Appium, Detox
- `🟡 MEDIUM` Visual testing — Percy, Applitools

### CI/CD Integration

- `🔴 CRITICAL` Running tests in CI pipelines — GitHub Actions, GitLab CI
- `🔴 CRITICAL` Test reporting — JUnit XML, Allure, HTML reports
- `🟠 HIGH` Parallel test execution — sharding, containers
- `🟡 MEDIUM` Flaky test management — retries, quarantining

### Test Strategy

- `🔴 CRITICAL` Risk-based testing — prioritizing tests by impact
- `🔴 CRITICAL` Test pyramid — unit vs integration vs E2E balance
- `🔴 CRITICAL` Regression strategy — what to automate vs keep manual
- `🟠 HIGH` Contract testing — Pact, verifying API contracts
- `🟡 MEDIUM` Performance test strategy — k6, JMeter, Gatling

### Programming

- `🔴 CRITICAL` JavaScript / TypeScript or Python for test code
- `🔴 CRITICAL` Version control — branching strategies for test repos
- `🟠 HIGH` Writing maintainable test code — DRY, readable assertions

### Shift-Left Testing

- `🟠 HIGH` Reviewing requirements and specs for testability
- `🟠 HIGH` Static analysis — ESLint, SonarQube, CodeQL
- `🟢 LOW` Mutation testing — verifying test quality

### Process & Culture

- `🔴 CRITICAL` Bug triage — severity, priority, root cause categorization
- `🔴 CRITICAL` Test metrics — pass/fail rates, coverage, defect density
- `🟠 HIGH` Agile testing practices — BDD, Specification by Example
- `🟠 HIGH` Mentoring junior QA engineers — code review, test design
