# Senior Full-Stack Developer

## Description

What a senior full-stack developer should know — end-to-end architecture, system design, technology decisions, and leading cross-functional technical initiatives.

## Prerequisites

- [Mid Full-Stack Developer](../mid/full-stack.md)

## Learning Path

### Architecture

- `🔴 CRITICAL` Full-stack architecture patterns — BFF, API gateway, monorepo
- `🔴 CRITICAL` Deployment architectures — serverless, containers, edge
- `🔴 CRITICAL` Data flow design — client → API → services → data stores
- `🔴 CRITICAL` Authentication and authorization architecture
- `🟠 HIGH` Multi-tenant system design
- `🟠 HIGH` Real-time architecture — WebSocket clusters, pub/sub

### System Design

- `🔴 CRITICAL` Designing for scale — caching, CDN, read replicas
- `🔴 CRITICAL` Database selection — relational vs NoSQL vs specialized stores
- `🔴 CRITICAL` API design — REST, GraphQL, gRPC — choosing the right one
- `🟠 HIGH` Event-driven architecture — Kafka, RabbitMQ, event stores
- `🟠 HIGH` CI/CD pipeline design — build, test, deploy strategy

### Performance

- `🔴 CRITICAL` End-to-end performance optimization — frontend to database
- `🔴 CRITICAL` Performance budgeting and monitoring
- `🔴 CRITICAL` Load testing — designing realistic benchmarks
- `🟠 HIGH` Profiling — identifying bottlenecks across the stack

### Security

- `🔴 CRITICAL` OWASP Top 10 — prevention and detection
- `🔴 CRITICAL` Secure software development lifecycle
- `🟠 HIGH` Penetration testing basics
- `🟠 HIGH` Compliance requirements — data protection, auditing

### Quality

- `🔴 CRITICAL` Testing strategy — unit, integration, E2E, performance
- `🔴 CRITICAL` Code review standards — security, performance, maintainability
- `🟠 HIGH` Static analysis and linting at scale
- `🟢 LOW` Property-based and contract testing

### Leadership

- `🔴 CRITICAL` Leading technical initiatives across frontend and backend teams
- `🔴 CRITICAL Cross-team communication — product, design, infrastructure
- `🔴 CRITICAL Technical decision documentation — RFCs, architecture decision records
- `🟠 HIGH Hiring and interviewing — evaluating full-stack candidates
