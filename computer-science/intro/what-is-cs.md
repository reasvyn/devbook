# What Is Computer Science

## Description

Computer science is the study of computation — what can be computed, how efficiently, and at what cost. For developers, CS provides the theoretical toolkit that separates "it works" from "it works well at scale."

## Content / Material

### Computation and Abstraction

At its core, CS is about layers of abstraction:

```
Hardware → Machine code → OS → High-level language → Framework → Application
```

Each layer hides complexity from the one above. Understanding the lower layers makes you a better engineer at every level above.

### Core Pillars of CS

| Pillar | Why Developers Need It |
|--------|----------------------|
| **Data structures** | Choose the right container for the job — arrays, trees, hash maps, graphs |
| **Algorithms** | Sort, search, traverse, and optimize — the recipes of computation |
| **Complexity theory** | Big-O analysis — predict performance before you run it |
| **Operating systems** | Processes, threads, memory, file systems — what your code runs on |
| **Compilers** | How human-readable code becomes machine instructions |
| **Databases** | Storage, indexing, query optimization — managing data at scale |
| **Architecture** | CPU, cache, memory hierarchy — write hardware-aware code |

### Theory vs. Practice

CS theory gives you the **why**. Practical programming gives you the **how**.

| Theory | Practice |
|--------|----------|
| Big-O notation | Profiling and optimization |
| B-tree theory | Database indexing |
| Graph algorithms | GPS routing, social networks |
| State machines | Regular expressions, protocol handlers |

## Glossary

| Term | Definition |
|------|------------|
| Big-O notation | A measure of algorithmic complexity in terms of input size |
| Abstraction | Hiding implementation details behind a clean interface |
| Computation | The process of performing mathematical or logical operations |
