# Senior DevOps Engineer

## Description

What a senior DevOps engineer should know — platform engineering, SRE practices, cloud architecture, organizational reliability strategy, and infrastructure leadership.

## Prerequisites

- [Mid DevOps Engineer](../mid/devops.md)

## Learning Path

### Platform Engineering

- `[must-know]` Internal developer platforms — designing golden paths
- `[must-know]` Self-service infrastructure — letting teams provision their own
- `[must-know]` Kubernetes at scale — multi-cluster, operators, custom controllers
- `[good-to-know]` Service mesh — Istio, Linkerd, mTLS, traffic policies
- `[good-to-know]` GitOps — ArgoCD, Flux, promoting changes declaratively

### SRE Practices

- `[must-know]` Error budgets — defining SLOs, managing reliability trade-offs
- `[must-know]` Toil elimination — measuring and reducing operational work
- `[must-know]` Incident response — leading major incidents, post-mortem culture
- `[good-to-know]` Chaos engineering — Game Days, Litmus, chaos mesh
- `[nice-to-have]` Capacity planning — modeling growth, right-sizing

### Cloud Architecture

- `[must-know]` Multi-region and multi-cloud strategies
- `[must-know]` Network architecture — VPC design, peering, transit gateways
- `[must-know]] Cost optimization — FinOps, reserved instances, spot instances
- `[good-to-know]` Serverless at scale — Lambda, Cloud Functions, Fargate
- `[good-to-know]` Infrastructure policy as code — OPA, Sentinel

### Security

- `[must-know]` Zero-trust architecture — beyond corporate VPN
- `[must-know]` Supply chain security — SBOM, image signing, dependency verification
- `[must-know]` Compliance automation — SOC2, HIPAA, PCI audit readiness
- `[good-to-know]` Penetration testing — infrastructure security assessments

### Observability at Scale

- `[must-know]` Metrics, logs, and traces — correlating signals
- `[must-know]` Building dashboards that tell a story — actionable vs vanity metrics
- `[must-know]` Alert fatigue — tuning alerts, meaningful paging
- `[good-to-know]` OpenTelemetry — standardizing instrumentation
- `[nice-to-have]` AIOps — anomaly detection, automated root cause analysis

### Leadership & Mentoring

- `[must-know]` Building DevOps culture — breaking down silos, shared ownership
- `[must-know]` Mentoring mid-level DevOps engineers
- `[must-know]] Technical proposals — infrastructure RFCs, migration plans
- `[good-to-know]` Vendor evaluation — cloud providers, tools, platforms
- `[good-to-know]` Budget and procurement — capacity costs, tool licensing
