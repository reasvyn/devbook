# Senior DevOps Engineer

## Description

What a senior DevOps engineer should know — platform engineering, SRE practices, cloud architecture, organizational reliability strategy, and infrastructure leadership.

## Prerequisites

- [Mid DevOps Engineer](../mid/devops.md)

## Learning Path

### Platform Engineering

- `🔴 CRITICAL` Internal developer platforms — designing golden paths
- `🔴 CRITICAL` Self-service infrastructure — letting teams provision their own
- `🔴 CRITICAL` Kubernetes at scale — multi-cluster, operators, custom controllers
- `🟠 HIGH` Service mesh — Istio, Linkerd, mTLS, traffic policies
- `🟠 HIGH` GitOps — ArgoCD, Flux, promoting changes declaratively

### SRE Practices

- `🔴 CRITICAL` Error budgets — defining SLOs, managing reliability trade-offs
- `🔴 CRITICAL` Toil elimination — measuring and reducing operational work
- `🔴 CRITICAL` Incident response — leading major incidents, post-mortem culture
- `🟠 HIGH` Chaos engineering — Game Days, Litmus, chaos mesh
- `🟡 MEDIUM` Capacity planning — modeling growth, right-sizing

### Cloud Architecture

- `🔴 CRITICAL` Multi-region and multi-cloud strategies
- `🔴 CRITICAL` Network architecture — VPC design, peering, transit gateways
- `🔴 CRITICAL Cost optimization — FinOps, reserved instances, spot instances
- `🟠 HIGH` Serverless at scale — Lambda, Cloud Functions, Fargate
- `🟠 HIGH` Infrastructure policy as code — OPA, Sentinel

### Security

- `🔴 CRITICAL` Zero-trust architecture — beyond corporate VPN
- `🔴 CRITICAL` Supply chain security — SBOM, image signing, dependency verification
- `🔴 CRITICAL` Compliance automation — SOC2, HIPAA, PCI audit readiness
- `🟠 HIGH` Penetration testing — infrastructure security assessments

### Observability at Scale

- `🔴 CRITICAL` Metrics, logs, and traces — correlating signals
- `🔴 CRITICAL` Building dashboards that tell a story — actionable vs vanity metrics
- `🔴 CRITICAL` Alert fatigue — tuning alerts, meaningful paging
- `🟠 HIGH` OpenTelemetry — standardizing instrumentation
- `🟡 MEDIUM` AIOps — anomaly detection, automated root cause analysis

### Leadership & Mentoring

- `🔴 CRITICAL` Building DevOps culture — breaking down silos, shared ownership
- `🔴 CRITICAL` Mentoring mid-level DevOps engineers
- `🔴 CRITICAL Technical proposals — infrastructure RFCs, migration plans
- `🟠 HIGH` Vendor evaluation — cloud providers, tools, platforms
- `🟠 HIGH` Budget and procurement — capacity costs, tool licensing
