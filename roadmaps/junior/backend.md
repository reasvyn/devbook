# Junior Backend Developer

## Description

What a junior backend developer should know — server-side fundamentals, API basics, database operations, and shipping simple backend features.

## Prerequisites

- Basic programming knowledge — variables, functions, control flow

## Learning Path

### Language Fundamentals

Pick **one** primary language:
- `[must-know]` **Node.js / TypeScript** — modules, async/await, file system, HTTP module
- `[must-know]` **Python** — functions, modules, virtual environments, pip
- `[must-know]` **Go** — packages, structs, interfaces, goroutines basics
- `[must-know]` **Java / C#** — classes, interfaces, dependency injection, build tools (Maven/Gradle)

### Database Basics

- `[must-know]` SQL — SELECT, INSERT, UPDATE, DELETE, WHERE, JOINs
- `[must-know]` Database design — tables, columns, primary keys, foreign keys
- `[good-to-know]` Connecting to a database from code — connection strings, querying
- `[nice-to-have]` Basic indexing — what it is and why it matters

### API Development

- `[must-know]` HTTP methods — GET, POST, PUT, DELETE, PATCH
- `[must-know]` RESTful API design — routes, status codes, request/response
- `[must-know]` Request handling — query params, path params, request body
- `[must-know]` JSON — parsing, serialization, content-type headers
- `[good-to-know]` Input validation — sanitizing and validating request data
- `[good-to-know]` Error handling — meaningful error messages, error status codes

### Version Control

- `[must-know]` Git basics — add, commit, push, pull, branch, merge
- `[must-know]` Pull requests — creating and responding to review feedback
- `[good-to-know]` .gitignore — what to exclude from version control

### Testing

- `[good-to-know]` Writing unit tests for business logic
- `[nice-to-have]` API testing with Supertest, pytest, or httptest

### Security Basics

- `[must-know]` Never trust user input — validation, sanitization
- `[must-know]` SQL injection — what it is and how to prevent it (parameterized queries)
- `[good-to-know]` Password hashing — bcrypt, never store plaintext
- `[good-to-know]` HTTPS — why it matters for APIs

### Deployment Basics

- `[good-to-know]` Environment variables — storing config outside code
- `[nice-to-have]` Deploying a simple API to a cloud platform (Render, Railway, Fly.io)

### Soft Skills

- `[must-know]` Reading and understanding logs
- `[must-know]` Asking for help effectively — what to include in a question
- `[good-to-know]` Documenting APIs — README, Swagger basics
