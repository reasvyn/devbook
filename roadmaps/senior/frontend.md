# Senior Frontend Developer

## Description

What a senior frontend developer should know — architecture decisions, design systems, performance optimization, accessibility compliance, and technical leadership.

## Prerequisites

- [Mid Frontend Developer](../mid/frontend.md)

## Learning Path

### Architecture & Design

- `[must-know]` Frontend architecture — micro-frontends, module federation
- `[must-know]` Design systems — component libraries, tokens, Storybook
- `[must-know]` State management architecture — when to use what (context, Redux, query tools)
- `[must-know]` Monorepo strategies — Nx, Turborepo
- `[good-to-know]` Web Components and framework-agnostic design
- `[good-to-know]` Edge rendering and CDN strategies

### Performance at Scale

- `[must-know]` Performance budgets — setting and enforcing limits
- `[must-know]` Core Web Vitals deep dive — optimizing for real users (RUM)
- `[must-know]` Rendering strategies — CSR, SSR, SSG, ISR, streaming SSR
- `[must-know]` Bundle optimization — code splitting, tree shaking, dynamic imports
- `[good-to-know]` Web Workers and off-main-thread processing
- `[good-to-know]` Image and video optimization pipelines

### Accessibility

- `[must-know]` WCAG 2.2 compliance — AA, AAA standards
- `[must-know]` Screen reader testing — VoiceOver, NVDA, JAWS
- `[must-know]` Keyboard navigation and focus management
- `[good-to-know]` Accessibility automation — axe-core, Lighthouse CI

### Testing Strategy

- `[must-know]` Testing strategy definition — what to test at each level
- `[must-know]` E2E testing at scale — Playwright, Cypress, visual regression
- `[good-to-know]` Performance testing — Lighthouse CI, WebPageTest
- `[nice-to-have]` Fuzzing and property-based testing for UI

### Mobile & Cross-Platform

- `[good-to-know]` React Native or Flutter for mobile
- `[good-to-know]` PWA — offline support, service workers, manifest
- `[nice-to-have]` Desktop — Electron, Tauri

### Code Quality & Mentoring

- `[must-know]` Code review at scale — reviewing architecture, not just syntax
- `[must-know]` Mentoring mid and junior developers
- `[must-know]] Technical decision records — documenting architecture decisions
- `[good-to-know]` Onboarding — creating effective onboarding for new frontend engineers

### Technical Leadership

- `[must-know]` Cross-team collaboration — aligning frontend practices across org
- `[must-know]` Technical proposals — RFCs, ADRs, design docs
- `[must-know]` Estimation and planning — scoping large frontend initiatives
- `[good-to-know]` Vendor evaluation — choosing frameworks, tools, platforms
