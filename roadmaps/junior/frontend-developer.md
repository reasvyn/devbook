# Junior Frontend Developer

## Description

What a junior frontend developer should know — the fundamentals that every entry-level frontend engineer needs to be productive on a real codebase.

## Prerequisites

- Basic computer literacy — files, folders, terminal

## Learning Path

### HTML & CSS

- `🔴 CRITICAL` [Semantic HTML](../../programming/html/index.md) — elements, document structure, forms
- `🔴 CRITICAL` [CSS Selectors](../../programming/css/selectors.md) — type, class, ID, attribute selectors
- `🔴 CRITICAL` [Box Model](../../programming/css/box-model.md) — content, padding, border, margin
- `🔴 CRITICAL` [Cascade & Specificity](../../programming/css/cascade-and-specificity.md) — how CSS resolves conflicts
- `🔴 CRITICAL` [Flexbox](../../programming/css/flexbox.md) — one-dimensional layout
- `🔴 CRITICAL` [CSS Grid](../../programming/css/grid.md) — two-dimensional layout
- `🔴 CRITICAL` [Responsive Design](../../programming/css/media-queries.md) — media queries, mobile-first
- `🔴 CRITICAL` [CSS Positioning](../../programming/css/positioning.md) — static, relative, absolute, fixed, sticky
- `🟠 HIGH` [CSS Transitions & Animations](../../programming/css/transitions-and-animations.md) — movement and interactivity
- `🟢 LOW` [Sass Basics](../../programming/css/sass-basics.md) — variables, nesting, mixins

### JavaScript

- `🔴 CRITICAL` [Programming Fundamentals](../../programming/fundamentals/index.md) — variables, types, operators, control flow
- `🔴 CRITICAL` [Functions](../../programming/fundamentals/functions.md) — declarations, expressions, arrow functions, scope
- `🔴 CRITICAL` [Collections](../../programming/fundamentals/collections.md) — arrays, objects, creation, iteration, methods (map, filter, reduce)
- `🔴 CRITICAL` [DOM Manipulation](../../programming/js-ts/dom-manipulation.md) — selecting, creating, modifying elements
- `🔴 CRITICAL` [Events](../../programming/js-ts/events.md) — addEventListener, event object, delegation
- `🔴 CRITICAL` [Fetch API](../../programming/js-ts/fetch-api.md) — GET and POST requests, handling responses
- `🔴 CRITICAL` [Promises & Async/Await](../../programming/js-ts/promises-async-await.md) — async patterns
- `🔴 CRITICAL` [Debugging with DevTools](../../programming/js-ts/debugging-devtools.md) — browser DevTools
- `🟠 HIGH` [ES6+ Features](../../programming/js-ts/es6-plus.md) — destructuring, spread/rest, template literals
- `🟠 HIGH` [Values & Types](../../programming/js-ts/values-and-types.md) — type coercion, equality, type checking
- `🟠 HIGH` [Functions in Depth](../../programming/js-ts/functions-in-depth.md) — closures, scope, this, higher-order functions
- `🟠 HIGH` [Objects & Prototypes](../../programming/js-ts/objects-and-prototypes.md) — prototypes, classes, inheritance
- `🟠 HIGH` [Web Storage & Cookies](../../programming/js-ts/web-storage.md) — localStorage, sessionStorage, cookies
- `🟠 HIGH` [JavaScript Modules & Tooling](../../programming/js-ts/modules-and-tooling.md) — ESM, npm, bundlers
- `🟡 MEDIUM` [TypeScript Basics](../../programming/js-ts/typescript-basics.md) — types, interfaces, generics

### Version Control

- `🔴 CRITICAL` [Git basics](../../software/version-control/git-basics.md) — init, add, commit, push, pull, status, log
- `🔴 CRITICAL` [Branching](../../software/version-control/branching-and-merging.md) — create, switch, merge branches
- `🟠 HIGH` Pull requests — creating, reviewing, resolving simple conflicts

### Build Tools & Package Managers

- `🔴 CRITICAL` npm / yarn — installing, updating, removing packages
- `🔴 CRITICAL` Running scripts from package.json
- `🟠 HIGH` Module imports and exports (ESM)

### Frameworks

- `🔴 CRITICAL` React — components, JSX, props, state, useEffect, useState
- `🟠 HIGH` Conditional rendering, lists and keys
- `🟠 HIGH` Handling forms and events in React

### Testing

- `🟠 HIGH` [What testing is and why it matters](../../software/software-testing/intro/what-is-software-testing.md)
- `🟡 MEDIUM` [Writing simple unit tests](../../software/software-testing/test-types.md) with Vitest or Jest

### Soft Skills

- `🔴 CRITICAL` Asking good questions — when and how to ask for help
- `🔴 CRITICAL` Reading documentation effectively
- `🟠 HIGH` Code review etiquette — giving and receiving feedback
