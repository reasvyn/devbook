# Junior Frontend Developer

## Description

What a junior frontend developer should know — the fundamentals that every entry-level frontend engineer needs to be productive on a real codebase.

## Prerequisites

- Basic computer literacy — files, folders, terminal

## Learning Path

### HTML & CSS

- `[must-know]` Semantic HTML — elements, document structure, forms
- `[must-know]` CSS selectors, box model, specificity, cascade
- `[must-know]` Flexbox and CSS Grid basics
- `[must-know]` Responsive design — media queries, mobile-first
- `[must-know]` CSS positioning — static, relative, absolute, fixed, sticky
- `[good-to-know]` CSS transitions and simple animations
- `[nice-to-have]` Sass basics — variables, nesting

### JavaScript

- `[must-know]` Variables, types, operators, control flow
- `[must-know]` Functions — declarations, expressions, arrow functions, scope
- `[must-know]` Arrays and objects — creation, iteration, methods (map, filter, reduce)
- `[must-know]` DOM manipulation — selecting, creating, modifying elements
- `[must-know]` Events — addEventListener, event object, delegation
- `[must-know]` Fetch API — GET and POST requests, handling responses
- `[must-know]` Promises and async/await basics
- `[must-know]` Debugging — browser DevTools (Elements, Console, Network, Sources)
- `[good-to-know]` ES6+ features — destructuring, spread/rest, template literals
- `[good-to-know]` Error handling — try/catch, throwing errors

### Version Control

- `[must-know]` Git basics — init, add, commit, push, pull, status, log
- `[must-know]` Branching — create, switch, merge branches
- `[good-to-know]` Pull requests — creating, reviewing, resolving simple conflicts

### Build Tools & Package Managers

- `[must-know]` npm / yarn — installing, updating, removing packages
- `[must-know]` Running scripts from package.json
- `[good-to-know]` Module imports and exports (ESM)

### Frameworks

- `[must-know]` React — components, JSX, props, state, useEffect, useState
- `[good-to-know]` Conditional rendering, lists and keys
- `[good-to-know]` Handling forms and events in React

### Testing

- `[good-to-know]` What testing is and why it matters
- `[nice-to-have]` Writing simple unit tests with Vitest or Jest

### Soft Skills

- `[must-know]` Asking good questions — when and how to ask for help
- `[must-know]` Reading documentation effectively
- `[good-to-know]` Code review etiquette — giving and receiving feedback
