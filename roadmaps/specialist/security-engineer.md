# Security Engineer

## Description

What a security engineer should know — application security, infrastructure security, threat modeling, incident response, and building security culture across an organization.

## Prerequisites

- Senior-level experience in software engineering or infrastructure

## Learning Path

### Application Security

- `[must-know]` OWASP Top 10 — detection, prevention, remediation
- `[must-know]` Secure code review — finding vulnerabilities in code
- `[must-know]` SAST and DAST — integrating security scanning into CI/CD
- `[must-know]` Authentication and authorization — OAuth2, OIDC, SAML, RBAC, ABAC
- `[good-to-know]` Secure API design — rate limiting, input validation, CORS, CSRF
- `[good-to-know]` Cryptography — hashing, encryption, signing, key management
- `[nice-to-have]` Bug bounty program management

### Infrastructure Security

- `[must-know]` Cloud security — IAM policies, security groups, network ACLs
- `[must-know]` Container security — image scanning, runtime security, seccomp
- `[must-know]` Kubernetes security — RBAC, pod security policies, network policies
- `[must-know]` Secrets management — Vault, AWS Secrets Manager, encryption at rest
- `[good-to-know]` Zero-trust architecture — beyond VPN, mTLS, identity-aware proxies
- `[good-to-know]` Infrastructure compliance — CIS benchmarks, hardening guides

### Security Operations

- `[must-know]` Vulnerability management — scanning, triage, patching lifecycle
- `[must-know]` Incident response — detection, containment, eradication, recovery
- `[must-know]` Forensics — log analysis, memory analysis, timeline reconstruction
- `[good-to-know]` SIEM — Splunk, ELK, Sentinel, detection rules
- `[good-to-know]` Threat intelligence — threat feeds, IOCs, adversary TTPs

### Threat Modeling

- `[must-know]` Threat modeling frameworks — STRIDE, PASTA, LINDDUN
- `[must-know]` Attack tree and data flow diagram analysis
- `[must-know]` Risk assessment — likelihood vs impact, risk scoring
- `[good-to-know]` Privacy impact assessments — data handling, PII

### Governance & Compliance

- `[must-know]` Security frameworks — NIST, ISO 27001, SOC 2
- `[must-know]` Data protection regulations — GDPR, CCPA, HIPAA, PCI-DSS
- `[must-know]` Security policies — developing and enforcing across org
- `[good-to-know]` Vendor security assessment — third-party risk management
- `[nice-to-have]` Penetration testing — scoping, executing, reporting

### Culture & Leadership

- `[must-know]` Security awareness training — phishing, social engineering, best practices
- `[must-know]` Building security champions program — embedding security in engineering
- `[must-know]` Communicating risk to non-technical stakeholders
- `[good-to-know]` Security metrics — measuring effectiveness of security program
- `[good-to-know]` Hiring and building a security team
