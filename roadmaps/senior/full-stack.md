# Senior Full-Stack Developer

## Description

What a senior full-stack developer should know — end-to-end architecture, system design, technology decisions, and leading cross-functional technical initiatives.

## Prerequisites

- [Mid Full-Stack Developer](../mid/full-stack.md)

## Learning Path

### Architecture

- `[must-know]` Full-stack architecture patterns — BFF, API gateway, monorepo
- `[must-know]` Deployment architectures — serverless, containers, edge
- `[must-know]` Data flow design — client → API → services → data stores
- `[must-know]` Authentication and authorization architecture
- `[good-to-know]` Multi-tenant system design
- `[good-to-know]` Real-time architecture — WebSocket clusters, pub/sub

### System Design

- `[must-know]` Designing for scale — caching, CDN, read replicas
- `[must-know]` Database selection — relational vs NoSQL vs specialized stores
- `[must-know]` API design — REST, GraphQL, gRPC — choosing the right one
- `[good-to-know]` Event-driven architecture — Kafka, RabbitMQ, event stores
- `[good-to-know]` CI/CD pipeline design — build, test, deploy strategy

### Performance

- `[must-know]` End-to-end performance optimization — frontend to database
- `[must-know]` Performance budgeting and monitoring
- `[must-know]` Load testing — designing realistic benchmarks
- `[good-to-know]` Profiling — identifying bottlenecks across the stack

### Security

- `[must-know]` OWASP Top 10 — prevention and detection
- `[must-know]` Secure software development lifecycle
- `[good-to-know]` Penetration testing basics
- `[good-to-know]` Compliance requirements — data protection, auditing

### Quality

- `[must-know]` Testing strategy — unit, integration, E2E, performance
- `[must-know]` Code review standards — security, performance, maintainability
- `[good-to-know]` Static analysis and linting at scale
- `[nice-to-have]` Property-based and contract testing

### Leadership

- `[must-know]` Leading technical initiatives across frontend and backend teams
- `[must-know]] Cross-team communication — product, design, infrastructure
- `[must-know]] Technical decision documentation — RFCs, architecture decision records
- `[good-to-know]] Hiring and interviewing — evaluating full-stack candidates
