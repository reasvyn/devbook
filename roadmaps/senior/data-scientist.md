# Senior Data Scientist

## Description

What a senior data scientist should know — advanced modeling, experiment design, ML pipeline architecture, stakeholder communication, and driving data-informed decisions.

## Prerequisites

- Foundational statistics and programming (equivalent of university-level coursework)

## Learning Path

### Advanced Modeling

- `[must-know]` Supervised learning — ensemble methods (XGBoost, LightGBM, Random Forest)
- `[must-know]` Unsupervised learning — clustering, dimensionality reduction (PCA, t-SNE)
- `[must-know]` Time series forecasting — ARIMA, Prophet, LSTMs
- `[must-know]` Deep learning — neural networks, CNNs, RNNs, transformers
- `[good-to-know]` Bayesian methods — probabilistic programming, PyMC
- `[good-to-know]` Causal inference — A/B testing, instrumental variables, difference-in-differences
- `[nice-to-have]` Reinforcement learning — MDP, Q-learning, policy gradients

### Experiment Design & Causal Reasoning

- `[must-know]` A/B testing — sample size, power, multiple testing correction
- `[must-know]` Experiment design — randomization, stratification, guarding metrics
- `[must-know]` Interpreting results — practical vs statistical significance
- `[good-to-know]` Online controlled experiments — ramp-up, holdout groups
- `[good-to-know]` Causal inference methods — propensity score matching, DiD

### ML Engineering

- `[must-know]` Feature engineering — feature stores, encoding, scaling
- `[must-know]` Model evaluation — cross-validation, metrics, bias/variance trade-off
- `[must-know]` MLOps — model versioning, deployment, monitoring
- `[good-to-know]` Data pipelines — Airflow, Prefect, dbt
- `[good-to-know]` Model serving — REST endpoints, batch inference, edge deployment
- `[nice-to-have]` Distributed training — Spark ML, Ray, Horovod

### Data Infrastructure

- `[must-know]` SQL for analytics — complex aggregations, window functions, CTEs
- `[must-know]` Data warehousing — BigQuery, Snowflake, Redshift
- `[good-to-know]` Data lake vs warehouse vs lakehouse
- `[good-to-know]` Streaming data — Kafka, Kinesis, real-time features

### Communication

- `[must-know]] Translating business questions into analytical problems
- `[must-know]] Communicating uncertainty — confidence intervals, error margins
- `[must-know]] Data visualization — telling stories with charts (Matplotlib, ggplot, Tableau)
- `[good-to-know]] Working with product and engineering — scoping, prioritization
- `[good-to-know]] Stakeholder management — getting buy-in for data initiatives

### Ethics & Governance

- `[must-know]` Bias in ML — detecting and mitigating algorithmic bias
- `[must-know]` Data privacy — GDPR, CCPA, data anonymization
- `[good-to-know]` Model fairness — fairness metrics, disparate impact
- `[good-to-know]` Explainability — SHAP, LIME, interpretable ML

### Leadership

- `[must-know]] Defining data science roadmap — prioritizing high-impact problems
- `[must-know]] Mentoring junior and mid data scientists
- `[good-to-know]] Building data culture — data literacy across the organization
- `[good-to-know]] Cross-functional collaboration — data engineering, ML engineering, product
