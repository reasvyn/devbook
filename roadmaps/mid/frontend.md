# Mid Frontend Developer

## Description

What a mid-level frontend developer should know — independence, deeper framework knowledge, testing, performance awareness, and mentoring juniors.

## Prerequisites

- [Junior Frontend Developer](../junior/frontend.md)

## Learning Path

### HTML & CSS

- `[must-know]` Accessibility — ARIA labels, keyboard navigation, screen reader testing
- `[must-know]` CSS custom properties, calc(), clamp()
- `[good-to-know]` CSS methodologies — BEM, utility-first (Tailwind)
- `[good-to-know]` Container queries, subgrid, advanced layouts
- `[nice-to-have]` Scalable CSS architecture

### JavaScript

- `[must-know]` Closures, hoisting, `this` binding
- `[must-know]` Prototypes and prototypal inheritance
- `[must-know]` Event loop — microtasks vs macrotasks, call stack
- `[must-know]` Modules — ESM vs CommonJS, dynamic imports
- `[good-to-know]` TypeScript — interfaces, unions, generics, utility types
- `[good-to-know]` Web APIs — localStorage, sessionStorage, canvas, WebSocket
- `[good-to-know]` Design patterns — singleton, observer, factory, module

### Frameworks

- `[must-know]` React hooks in depth — useMemo, useCallback, useRef, custom hooks
- `[must-know]` State management — context, Zustand, Redux Toolkit
- `[must-know]` Client-side routing — React Router, nested routes, guards
- `[good-to-know]` Next.js — pages router, SSR, ISR
- `[good-to-know]` Performance — React.memo, virtualization (react-window), profiling
- `[nice-to-have]` Alternative framework — Vue or Svelte

### Testing

- `[must-know]` Unit testing — Vitest/Jest, testing utilities
- `[must-know]` Component testing — React Testing Library
- `[must-know]` Mocking — API calls, modules, timers
- `[good-to-know]` E2E testing — Playwright or Cypress
- `[nice-to-have]` Visual regression testing — Percy, Chromatic

### Build & Performance

- `[must-know]` Vite / Webpack — configuration, loaders, plugins
- `[must-know]` Core Web Vitals — LCP, FID, CLS
- `[must-know]` Image optimization — formats, lazy loading, responsive
- `[good-to-know]` Code splitting and lazy loading
- `[good-to-know]` Bundle analysis and tree shaking
- `[nice-to-have]` Caching — service workers, CDN configuration

### API Integration

- `[must-know]` REST — error handling, loading states, optimistic updates
- `[good-to-know]` GraphQL — queries, mutations, Apollo Client
- `[good-to-know]` Authentication — JWT, refresh tokens, session management

### DevOps Awareness

- `[good-to-know]` Docker — writing a Dockerfile for frontend apps
- `[good-to-know]` CI/CD — automated testing in pipelines
- `[nice-to-have]` Deployment — Vercel, Netlify, Cloudflare Pages

### Soft Skills

- `[must-know]` Mentoring juniors — code review, pairing, knowledge sharing
- `[must-know]` Estimation — breaking down work, realistic timelines
- `[good-to-know]` Cross-team communication — backend, design, product
