# Mid QA Engineer

## Description

What a mid-level QA engineer should know — test automation, CI integration, test strategy, and mentoring junior testers.

## Prerequisites

- [Junior QA Engineer](../junior/qa-engineer.md)

## Learning Path

### Test Automation

- `[must-know]` E2E automation — Playwright, Cypress, or Selenium
- `[must-know]` Page Object Model — structuring automation code
- `[must-know]` API automation — REST API testing with code (Supertest, pytest, REST Assured)
- `[good-to-know]` Data-driven testing — parameterized test cases
- `[good-to-know]` Mobile automation — Appium, Detox
- `[nice-to-have]` Visual testing — Percy, Applitools

### CI/CD Integration

- `[must-know]` Running tests in CI pipelines — GitHub Actions, GitLab CI
- `[must-know]` Test reporting — JUnit XML, Allure, HTML reports
- `[good-to-know]` Parallel test execution — sharding, containers
- `[nice-to-know]` Flaky test management — retries, quarantining

### Test Strategy

- `[must-know]` Risk-based testing — prioritizing tests by impact
- `[must-know]` Test pyramid — unit vs integration vs E2E balance
- `[must-know]` Regression strategy — what to automate vs keep manual
- `[good-to-know]` Contract testing — Pact, verifying API contracts
- `[nice-to-have]` Performance test strategy — k6, JMeter, Gatling

### Programming

- `[must-know]` JavaScript / TypeScript or Python for test code
- `[must-know]` Version control — branching strategies for test repos
- `[good-to-know]` Writing maintainable test code — DRY, readable assertions

### Shift-Left Testing

- `[good-to-know]` Reviewing requirements and specs for testability
- `[good-to-know]` Static analysis — ESLint, SonarQube, CodeQL
- `[nice-to-have]` Mutation testing — verifying test quality

### Process & Culture

- `[must-know]` Bug triage — severity, priority, root cause categorization
- `[must-know]` Test metrics — pass/fail rates, coverage, defect density
- `[good-to-know]` Agile testing practices — BDD, Specification by Example
- `[good-to-know]` Mentoring junior QA engineers — code review, test design
